## Dast

### What to do
1. Review the findings below and address each in priority order.
2. Create tracking issues for any items requiring multi-sprint effort.

### Findings

#### 🟠 HIGH — Missing Content-Security-Policy header
- **Location:** `next.config.js`
- **Details:** No Content-Security-Policy header found. CSP prevents XSS, clickjacking, and data injection attacks by declaring allowed content sources. Without it, injected scripts can execute freely. (checked across 0 config file(s) + 0 route(s))
- **Steps:** Add CSP in next.config.js or middleware: `Content-Security-Policy: default-src 'self'; script-src 'self' 'nonce-{nonce}'; frame-ancestors 'none';`. Use report-only mode first to tune. NIST control: SC-18, SI-10

#### 🟠 HIGH — Missing Strict-Transport-Security (HSTS) header
- **Location:** `next.config.js`
- **Details:** HSTS is not configured. Without it, browsers may accept HTTP connections before upgrading to HTTPS, enabling SSL-stripping attacks and cookie interception. (checked across 0 config file(s) + 0 route(s))
- **Steps:** Add `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload` to all responses. Enable at the CDN layer (Vercel, Cloudflare) in addition to app code. NIST control: SC-8, SC-23

#### 🟡 MEDIUM — Missing X-Frame-Options header (clickjacking)
- **Location:** `next.config.js`
- **Details:** X-Frame-Options or CSP frame-ancestors is not set. An attacker can embed this app in an iframe on a malicious site to trick users into clicking UI elements (clickjacking). (checked across 0 config file(s) + 0 route(s))
- **Steps:** Add `X-Frame-Options: DENY` or include `frame-ancestors 'none'` in your CSP. In Next.js: add to headers() in next.config.js. NIST control: SC-18

#### 🟡 MEDIUM — Missing X-Content-Type-Options header
- **Location:** `next.config.js`
- **Details:** Without `X-Content-Type-Options: nosniff`, browsers may sniff the content type of a response, potentially executing uploaded files as scripts (MIME confusion attacks). (checked across 0 config file(s) + 0 route(s))
- **Steps:** Add `X-Content-Type-Options: nosniff` to all responses. This is a one-liner in next.config.js headers(). NIST control: SI-10

#### 🟢 LOW — Missing Referrer-Policy header
- **Location:** `next.config.js`
- **Details:** Referrer-Policy is not set. By default, browsers send the full URL as Referer, which can leak sensitive paths/tokens to third-party analytics or CDNs. (checked across 0 config file(s) + 0 route(s))
- **Steps:** Add `Referrer-Policy: strict-origin-when-cross-origin` to your response headers. NIST control: SC-8

#### 🟢 LOW — Missing Permissions-Policy header
- **Location:** `next.config.js`
- **Details:** Permissions-Policy (formerly Feature-Policy) is not configured. This header restricts which browser features (camera, microphone, geolocation) the page can request. (checked across 0 config file(s) + 0 route(s))
- **Steps:** Add `Permissions-Policy: camera=(), microphone=(), geolocation=(), payment=()` to restrict unneeded browser APIs. NIST control: SC-18
